Implementada en arquitectura cliente/servidor tiene tres componentes básicos:

  • Servidor:
    • Encargado de proporcionar archivos desde un directorio predeterminado conocido como el directorio de publicación. Es a los archivos dentro de este directorio a los cuales los clientes tendrán acceso. Es un programa como iis, apache, ngnix, etc; que espera solicitudes de archivos de los clientes en los puertos estándar 80/tcp y 443/tcp
  • Cliente:
    • Es un programa encargado de solicitar o requerir archivos al servidor y mostrarlos (desplegarlos) en la ventana de la aplicación; de manera predeterminada recuperara o desplegara archivos HTML pero esta en la capacidad de mostrar cualquier archivo para el cual tenga el correspondiente conector(plugin) Ejemplos: de clientes web: Chrome, Firefox, iexplorer, edge, safari, opera, etc
  • Lenguaje de comunicación de red:
    • Es el lenguaje que utiliza el cliente y servidor para comunicarse, en este caso e lenguaje es el HTTP o protocolo de transferencia de hiper texto. Este lenguaje tiene derivado https, que realiza la comunicación de manera encriptada entre cliente y servidor utilizando la tecnología PKI(Public Key Infraestructure)

Tipos de Rutas

  • Rutas Absolutas
    Esta forma inicia con la carpeta de publicación representada por el carácter “/” y a continuación se nombran todos los nombres de directorios que se deben recorrer hasta llegar al archivo requerido
  • Rutas Relativas
    Esta forma toma en consideración la ubicación actual del archivo que incluye al otro archivo tomando en consideración las sgres abreviaturas:
    • . -------> Directorio actual
    • .. -------> Directorio padre

Tipos de listas

  • Listas Ordenadas (OL)
    Presenta la información identificada por un número , cada nodo de información es presentado como un ítem
  • Listas No Ordenadas (UL)
    Presenta la información sin numeración
  • Listas De Definicion (DL)
    Presenta la información identificada por un título (DT y cada información es mostrada como definition data(DD)
